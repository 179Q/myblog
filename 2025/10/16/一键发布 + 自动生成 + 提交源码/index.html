Hexo 一键发布 + 自动生成 + 提交源码

Created2025-10-16|Updated2025-10-16
|Post Views:

完全可以做到 一键发布 + 自动生成 + 提交源码，不需要你每次手动 hexo clean。

我给你详细说明一下：

1️⃣ Hexo 常规操作

一般 Hexo 更新文章需要三个步骤：

生成静态文件（hexo g 或 hexo generate）

将 source 里的 Markdown 转换成 HTML 静态文件，生成在 public 目录。

部署到 GitHub Pages（hexo d 或 hexo deploy）

将 public 文件夹内容推送到 Pages 仓库（比如 myblog.github.io）。

提交源码（可选）

将 source、主题、配置等推送到源码仓库（比如 myblog-source.git）。

注意：不一定每次都需要 hexo clean

hexo clean 是清理生成的 public 文件夹

仅在出现生成异常或页面显示混乱时需要

一般 hexo g 会覆盖已有的 public 文件，足够日常使用。

2️⃣ 一键操作方案

我们可以写一个 bat 文件，实现：

自动生成静态文件

自动部署到 GitHub Pages

自动提交源码仓库

示例 publish_blog.bat：

@echo off
REM ==========================
REM Hexo 一键生成 + 部署 + 提交源码
REM ==========================

cd /d %~dp0

echo ==========================
echo 生成博客静态文件…
echo ==========================
hexo g

echo ==========================
echo 部署到 GitHub Pages…
echo ==========================
hexo d

echo ==========================
echo 提交博客源码…
echo ==========================
REM 拉取远程仓库防止拒绝推送
git pull origin main –allow-unrelated-histories

REM 添加所有变更
git add .

REM 自动提交，带时间戳
for /f “tokens=1-4 delims=/ “ %%a in (‘date /t’) do set DATE=%%a-%%b-%%c
for /f “tokens=1-2 delims=: “ %%a in (‘time /t’) do set TIME=%%a-%%b
git commit -m “Update blog %DATE% %TIME%”

REM 推送到远程源码仓库
git push origin main

echo ==========================
echo 发布完成！
echo ==========================
pause

3️⃣ 使用方法

将 publish_blog.bat 放在 myblog 根目录（和 _config.yml 同级）。

每次修改文章或主题后，直接 双击这个 bat 文件。

它会顺序执行：

Hexo 生成静态文件

自动部署到 Pages

自动提交源码到 GitHub

整个流程你只需要点击一次，无需手动 clean 或分别执行 hexo g/d。

4️⃣ 小贴士

如果你担心 .deploy_git 或 public 文件夹冲突，可以在 bat 中加一行：

hexo clean

一般日常不需要，每次 hexo g 就能覆盖旧文件。

确保部署仓库和源码仓库已经配置好远程 SSH，否则会提示认证失败。
