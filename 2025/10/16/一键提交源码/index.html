Hexo 一键提交源码

添加或者修改完blog发布后，给一个bat，一键提交源码

明白了，你希望 写一个 Windows .bat 文件，在你更新或添加文章后，能 一键提交 Hexo 源码到 GitHub 源码仓库（myblog-source.git）。

下面我给你写一个示例脚本，包含所有安全操作：拉取远程、合并、提交、推送。

1️⃣ 创建批处理文件

在 myblog 根目录下，新建一个文件，例如：

push_blog.bat

内容如下：

@echo off
REM ==========================
REM 一键提交 Hexo 源码到 GitHub
REM ==========================

REM 进入博客目录
cd /d %~dp0

REM 生成最新博客静态文件（可选，如果只提交源码可注释）
REM hexo g

REM 拉取远程仓库并允许不相关历史合并
git pull origin main –allow-unrelated-histories

REM 添加所有变更
git add .

REM 提交变更，带时间戳
for /f “tokens=1-4 delims=/ “ %%a in (‘date /t’) do set DATE=%%a-%%b-%%c
for /f “tokens=1-2 delims=: “ %%a in (‘time /t’) do set TIME=%%a-%%b
git commit -m “Update blog %DATE% %TIME%”

REM 推送到远程仓库
git push origin main

echo.
echo ==========================
echo 提交完成！
echo ==========================
pause

2️⃣ 使用方法

把 push_blog.bat 放在 myblog 根目录下（和 _config.yml 同级）。

每次修改、添加文章或者主题、配置文件后，双击运行这个 bat 文件。

它会自动：

拉取远程仓库（防止被拒绝推送）

添加所有更改

自动生成带时间的提交信息

推送到 GitHub

⚠️ 注意事项

第一次推送：如果远程仓库有 README，可能会出现合并冲突，需要手动处理一次。

SSH 优先：确保你的远程仓库地址是 SSH（git@github.com:179Q/myblog-source.git），这样不会再提示用户名/密码。

只提交源码：.gitignore 中已忽略 public/、.deploy_git/ 等生成文件，保证推送的是源代码。
