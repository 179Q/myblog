Hexo 一键发布脚本

太好了 😎！那我们就来搞个 Hexo 一键发布脚本，让你以后写完文章后一行命令就能完成：
✅ 清理缓存
✅ 生成页面
✅ 上传 GitHub
✅ 自动打开你的博客网站

🚀 一、创建脚本文件

在你的 Hexo 博客根目录（myblog） 下，新建一个文件：

Windows：

文件名：deploy.bat

🧾 二、写入以下内容
@echo off
echo ========================================
echo 🚀 Hexo 博客一键发布工具 by ChatGPT
echo ========================================
echo.

:: 1. 清理缓存
echo 🔄 正在清理旧文件…
call hexo clean

:: 2. 生成静态文件
echo 🏗️ 正在生成静态页面…
call hexo generate

:: 3. 部署到 GitHub
echo 🌍 正在上传到 GitHub Pages…
call hexo deploy

:: 4. 自动打开网站
echo 🌐 打开博客网页…
start https://你的用户名.github.io/你的仓库名/

echo ✅ 部署完成！按任意键退出。
pause >nul

🧩 三、替换成你的信息

把最后一行的网址：

https://你的用户名.github.io/你的仓库名/

换成你的真实博客地址，例如：

https://179Q.github.io/myblog/

🧱 四、使用方法

以后每次写完文章，只要：

保存 Markdown 文件（在 source/_posts 下）；

双击运行 deploy.bat；

它会自动帮你：

清理缓存；

生成网站；

上传 GitHub；

打开你的博客。

整个过程只需几十秒 💨。

⚙️ 五、进阶版（自动保存+发布）

如果你想要更高级一点的效果（例如自动保存修改、检测变动就部署），
我可以帮你写一个 Node.js 自动监听脚本（持续监控 _posts 文件夹的变化，一旦你保存新文章，就自动执行 hexo clean && hexo g && hexo d）。

监听发布

非常好 👍 你要的是“写完保存 → 自动生成 → 自动发布到 GitHub Pages”的完整自动流程，对吧？
我们可以做到！下面是详细方案👇

🚀 一、实现目标

你只要在 source/_posts/ 下写文章（保存 .md 文件），
系统会自动执行：

hexo generate 生成静态网页；

hexo deploy 自动发布到 GitHub Pages。

🧩 二、方法一：用 Node.js 脚本 + chokidar 监听文件变化
1️⃣ 安装依赖

在你的 Hexo 根目录执行：

npm install chokidar –save-dev

2️⃣ 创建一个脚本文件

新建文件：auto-deploy.js

内容如下：

const { exec } = require(“child_process”);
const chokidar = require(“chokidar”);

console.log(“🔍 正在监听文章变化…”);

let deploying = false;

function deploy() {
if (deploying) return;
deploying = true;

console.log(“🚧 检测到改动，开始生成并部署…”);
exec(“hexo clean && hexo generate && hexo deploy”, (err, stdout, stderr) => {
if (err) {
console.error(“❌ 部署出错：”, err);
} else {
console.log(“✅ 部署完成！”);
console.log(stdout);
}
deploying = false;
});
}

// 监听 source/_posts 下的所有 md 文件
chokidar.watch(“source/_posts/**/*.md”, {
ignored: /[/\]./,
persistent: true,
}).on(“change”, deploy)
.on(“add”, deploy);

console.log(“🟢 已启动自动部署监听！”);

3️⃣ 运行脚本

在命令行执行：

node auto-deploy.js

此时它会一直运行，当你新增或修改 .md 文件时，Hexo 会自动：

清理缓存；

重新生成；

自动发布到 GitHub Pages。

🧩 三、方法二（更简单但需自己运行一次命令）

如果你不想一直开 Node.js，可以用：

hexo server -w

-w 参数会监听变化自动生成（但不会自动 deploy）。

所以结合：

hexo server -w
hexo deploy -g

可半自动化。

🧩 四、可选进阶：写成 npm script

在 package.json 添加：

“scripts”: {
“autodeploy”: “node auto-deploy.js”
}

然后你只需运行：

npm run autodeploy

搞定 ✅

要不要我帮你把这个自动部署脚本改成 Windows 双击可运行版（.bat 文件）？
这样你点一下就能自动监听 + 发布。
